# Visual QA Checklist

## Global Consistency

- [ ] 16:9 ratio is correct.
- [ ] Microsoft YaHei is used for Chinese text.
- [ ] Final deck matches the approved four-slide sample.
- [ ] Default or user-provided visual benchmark was respected.
- [ ] Navigation is consistent and lightweight.
- [ ] Main, supporting, accent, and neutral colors are consistent.
- [ ] Margins and page numbers are consistent.

## Typography

- [ ] Titles are readable from a distance.
- [ ] Body text is at least approximately 16 pt.
- [ ] References remain readable after rendering.
- [ ] No text overflow, clipping, or garbled Chinese.
- [ ] Bold and accent color are used sparingly.

## Figures and Tables

- [ ] Figures are high resolution and not distorted.
- [ ] Tables are legible in normal presentation view.
- [ ] No tiny source screenshot is retained merely to fill space.
- [ ] Native redraw and original thumbnail are not unnecessarily duplicated.
- [ ] Key result is emphasized without hiding context.
- [ ] Source note and original figure/table number are visible.
- [ ] Axes and proportions are not misleading.

## Layout and Hierarchy

- [ ] Every slide has one dominant conclusion.
- [ ] Reading order is apparent within three seconds.
- [ ] Intentional negative space is preserved where useful.
- [ ] Accidental empty areas do not reflect missing reasoning.
- [ ] Decoration does not carry more weight than evidence.
- [ ] No more than three cards appear on a normal slide.
- [ ] Consecutive slides do not repeat the same card grid.
- [ ] At least several of the five core page archetypes are used across the deck.
- [ ] Section-divider slides do not exceed four unless approved.
- [ ] Navigation never dominates the title or data.

## Deck-Level Rhythm

- [ ] A full-deck thumbnail montage was created.
- [ ] Repetitive layouts were identified and corrected.
- [ ] Color drift was corrected.
- [ ] Divider frequency was reviewed.
- [ ] The montage is visually consistent with the approved samples.

## Page-Specific Visual References

- [ ] Every final slide has a page-specific image2/image_gen full-slide reference.
- [ ] Each reference reflects the actual scientific logic and content hierarchy of that page.
- [ ] Result references follow one figure or one coherent panel + one interpretation block.
- [ ] The rebuilt slide was compared with the reference for hierarchy, spacing, and rhythm.
- [ ] No generic image was reused as the design solution for multiple unrelated slides.

## Generated Assets

- [ ] Transparent backgrounds are used when required.
- [ ] No white halo, watermark, pseudo-text, or garbled labels.
- [ ] Generated visual references are not used as flattened final slides.
