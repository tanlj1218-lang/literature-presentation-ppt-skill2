# Style and Content Architecture Approval QA

## Content Architecture

- [ ] Full-deck page blueprint exists.
- [ ] Every slide's content, scientific question, main takeaway, and previous/current/next logic are listed.
- [ ] Result figures have been split according to the one-figure/one-interpretation rule.
- [ ] The full page list was shown to the user.
- [ ] User approval of page order and depth is recorded, or the gate was explicitly waived.


## Four Required Samples

- [ ] Title slide is complete and editable.
- [ ] Introduction/theory sample is complete and editable.
- [ ] Methods sample is complete and editable.
- [ ] Results sample is complete and editable.
- [ ] All four slides are rendered as images for review.

## Benchmark Fidelity

- [ ] User-provided or previously approved template was inspected.
- [ ] If no template was supplied, the default visual benchmark was used.
- [ ] Navigation is lightweight.
- [ ] Palette, typography, margins, and citation style are coherent.
- [ ] Card usage is restrained.
- [ ] Figures and tables are readable.

## Approval Gate

- [ ] The user approved one visual direction.
- [ ] Approved design tokens were recorded.
- [ ] Full-deck production did not start before approval, unless the user explicitly waived this step.
- [ ] Later revisions preserve the approved style.
