# Image-to-PPT Reconstruction QA

- [ ] No full-slide generated reference is used as a flattened background.
- [ ] Formal text is rebuilt with native PowerPoint text boxes.
- [ ] Shapes, arrows, lines, navigation, and panels are native editable objects.
- [ ] Tables and charts are native when exact data are available.
- [ ] Complex paper figures use verified high-resolution source crops.
- [ ] Complex generated illustrations use transparent assets without white borders or pseudo-text.
- [ ] Coordinate mapping and layer order match the generated reference.
- [ ] The rebuilt slide preserves hierarchy, spacing, negative space, and reading path.
- [ ] Generated text errors were corrected using the slide specification.
- [ ] Citations and source notes are exact and readable.
- [ ] Result slide obeys one-figure/one-interpretation rule.
- [ ] Rendered PPT slide was compared side by side with its image2 reference.
- [ ] Text overflow, clipping, overlap, distortion, and edge cropping were fixed.
- [ ] All feasible elements remain editable.
